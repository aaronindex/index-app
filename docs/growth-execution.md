# checklist.md — INDEX Growth Execution

This checklist is derived directly from the 30-60-90 Growth Plan.  
Operate it weekly. No skipping gates.

---

## WEEK 1 — Frame + Instrument

- [ ] Define First Aha Moment
- [ ] Define Retention Intent Signal
- [ ] Confirm analytics:
  - [ ] Signup → first import
  - [ ] First Highlight / Decision surfaced
  - [ ] Day-7 return
- [ ] Build **INDEX Builds INDEX** demo project
  - [ ] Curated chats
  - [ ] Clean titles
  - [ ] Clear thinking arc
- [ ] Write internal doc: *Why INDEX exists*

**Gate:**  
☐ 30-second explanation  
☐ Self-evident demo

---

## WEEK 2 — Trusted Circle Validation

- [ ] Identify 10–20 high-signal users
- [ ] Personal outreach only
- [ ] Each user reviews demo project
- [ ] Each user imports one real item
- [ ] Capture structured feedback

**Gate:**  
☐ ≥5 retention-intent signals  
☐ ≥3 users describe same core benefit

---

## WEEK 3 — Sharpen + Soft Publicity

- [ ] Remove one non-essential product element
- [ ] Tighten onboarding around single use case
- [ ] Publish 1 long-form piece
- [ ] Publish 2 short POV posts
- [ ] Enable soft public signups

**Gate:**  
☐ ≥5 stranger signups  
☐ ≥1 inbound interest message

---

## WEEK 4 — Repeatability Test

- [ ] Repeat outreach with refined framing
- [ ] Observe onboarding friction
- [ ] Lock single repeatable story

**Gate:**  
☐ 30–50 total users  
☐ Clear ICP + problem + differentiation

---

## WEEKS 5–8 — Manual Growth Engine

### Weekly (repeat every week)

**Public**
- [ ] Publish 1 real INDEX artifact

**Direct**
- [ ] Send 20–30 personal outreaches

**Group**
- [ ] Host or join 1 small-group surface

**Product**
- [ ] Fix top friction point

**Weekly Gate:**  
☐ 10–20 new users  
☐ 3–5 meaningful conversations

---

## DAY-60 HARD CHECK

- [ ] Can clearly state how users are found
- [ ] Can articulate why they retain
- [ ] Can enroll users manually

If any unchecked → pause acquisition.

---

## WEEKS 9–12 — Compounding Phase

- [ ] Publish 3–5 foundational content pieces
- [ ] Polish demo project for self-serve clarity
- [ ] Tag users by use case and depth

---

## Paid Conversion Test

- [ ] Define free tier limits
- [ ] Define paid continuity unlock
- [ ] Observe upgrade attempts

**Gate:**  
☐ ≥2–5% upgrade attempts  
☐ Clear payment motivation language

---

## Capital + Ads Readiness

Do **not** proceed unless all are true:
- [ ] ≥300 users
- [ ] Clear retention curve
- [ ] First paid conversions
- [ ] Repeatable enrollment motion
