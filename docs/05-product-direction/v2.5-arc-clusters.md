# Arc Clusters Across Projects (Future Feature)

Arc clusters group structurally similar arcs across multiple projects.

This enables INDEX to identify broader structural themes in a user's thinking.

Example:

Projects:
- INDEX product
- Landing page
- Onboarding

Each may contain arcs related to launch preparation.

These could cluster into:

"Preparing INDEX v2 Launch"

---

## Purpose

Arc clusters allow the system to detect:

- structural convergence across projects
- broader thematic direction
- multi-project momentum

Clusters improve:

- Direction summaries
- Weekly digests
- Timeline interpretation

---

## Conceptual Model

project arcs
↓
embedding similarity
↓
cluster detection
↓
cluster labels

Clusters are not stored in the structural layer.

They are a semantic interpretation layer.

---

## Potential UI Surfaces

Direction block

Example:

Structure is concentrating around preparing the INDEX v2 launch.

Two arcs across separate projects contribute to this shift.

---

## Implementation Ideas

Option A — LLM clustering

Provide arc summaries and ask model to group them.

Option B — embedding clustering

Use cosine similarity on arc summaries.

Option C — hybrid

Embedding grouping → LLM label generation.
